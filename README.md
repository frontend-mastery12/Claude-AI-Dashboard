## 🧠 Overview

**Gym Pulse** is a fully functional, premium dark-mode dashboard built for gym operators to track members, revenue, class utilization, transactions, and performance analytics — all designed and generated by **Claude AI** from a single prompt.

This project proves that a well-structured instruction document beats dozens of scattered prompts. Instead of manually describing every button and layout choice, a detailed `context.md` file — covering design language, page structure, component patterns, and data contracts — was handed to Claude AI along with reference UI images. The result: a complete, functional, three-page dashboard generated in minutes, entirely on Claude's free plan.

No backend. No manual coding. Just one context file, three reference images, and one prompt.

## ✨ Key Features

### 📊 Dashboard Page
- **Hero KPI Row** — active members, weekly revenue, class utilization, and new leads at a glance
- **Activity Summary** — recent check-ins, session bookings, and coach availability
- **Workout Completion Metric** — progress ring / sparkline visualization
- **Revenue Overview** — compact live-updating area/line chart powered by Recharts
- **Member Pulse Cards** — top-performing trainers, new signups, and churn risk indicators
- **Responsive Grid** — 2–3 column layout on desktop, gracefully stacked on mobile

### 💳 Transactions Page
- **Full Transaction Table** — Date, Member, Plan, Amount, and Status columns
- **Row Selection** — checkbox-based selection with bulk action toolbar (Delete, Archive, Export)
- **Status Chips** — Complete, Pending, and Failed states with distinct accent colors
- **Payment Method Display** — shows exactly how each member paid
- **Search & Filter** — filter by transaction type, status, or date range
- **Deletion Feedback** — inline undo banner on delete, with bulk or one-by-one removal

### 📈 Analytics Page
- **Revenue Trend Chart** — monthly/quarterly performance via area or line chart
- **Conversion Funnel** — signup → trial activation → retention rate
- **Engagement Analytics** — workout sessions, class attendance, personal training uptake
- **Category Heatmap / Segmented Bars** — top workout categories and peak time slots
- **Comparison Cards** — current vs. prior period, target attainment, and forecast

### 🎨 UI & UX
- **Premium Dark Theme** — deep black canvas with layered card surfaces and a luminous green accent
- **Glassmorphism-Inspired Surfaces** — subtle gradients, soft borders, and layered depth
- **Micro-Interactions** — hover glow and scale effects on cards and nav items
- **Accessible by Design** — proper `aria` attributes and visible keyboard focus states
- **Fully Responsive** — consistent spacing and hierarchy from mobile to desktop

## 🎨 Design Language

| Token | Value |
|---|---|
| Primary canvas | `#0A0A0A` |
| Secondary surfaces | `#111111` |
| Accent (success) | `#7CFF6B` |
| Warning | `#F5B700` |
| Neutral | `#9CA3AF` |

A layered, premium dark-mode system built around high contrast typography, tight-but-breathable spacing, and a single luminous accent color used consistently across charts, buttons, and status indicators.

---

## 🏗️ Layout System

- **Sidebar** — fixed left navigation with brand mark, nav links, quick actions, and user profile; active route highlighted in accent green
- **Topbar** — semi-transparent header with page title, search, notifications, and profile access
- **Main Content Area** — responsive grid/stack layout hosting KPI cards, charts, tables, and lists with consistent page padding

## 🛠 Tech Stack

| Technology | Purpose |
|---|---|
| **React (Vite)** | Component-based UI, fast dev/build tooling |
| **Tailwind CSS** | Utility-first responsive styling with custom theme tokens |
| **React Router DOM** | Client-side routing across Dashboard, Transactions, and Analytics |
| **Recharts** | Dark-themed, responsive charts (area, line, bar, pie/radial) |

> ⚠️ **Frontend only** — no backend, no database, no authentication. All data is mock data defined in local JS files, structured to mirror a real API response shape.

## 🔀 Routing

| Route | Page |
|---|---|
| `/` | Dashboard |
| `/transactions` | Transactions |
| `/analytics` | Analytics |

Handled via React Router DOM, with route-driven active-state styling in the sidebar and dynamic page titles in the topbar.

---

## 💳 Transaction Data Contract

```js
transaction = {
  id: Number,
  date: String,
  member: String,
  service: String,
  paymentMethod: String,
  amount: Number,
  status: "Complete" | "Pending" | "Failed"
}
```

**State ownership:** transaction state lives entirely inside `Transactions.jsx`, which passes down handler props (`onToggleSelect`, `onDelete`, etc.) to purely presentational child components — no state duplication, no prop drilling beyond one level.

## 🚀 Getting Started

### Prerequisites
- Node.js v18+
- npm v9+

### Installation
```bash
# Clone the repository
git clone https://github.com/frontend-mastery12/Claude-AI-Dashboard.git

# Navigate into the project
cd Claude-AI-Dashboard

# Install dependencies
npm install

# Start the development server
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

### Build for Production
```bash
npm run build
npm run preview
```

## 🤖 How This Was Built

This entire dashboard was generated by **Claude AI** from a single, richly detailed prompt referencing:

1. A **1,000+ line `context.md`** file — defining design language, layout system, page-by-page specs, component patterns, data contracts, and Tailwind/Recharts conventions
2. **Three reference UI images** — Dashboard, Transactions, and Analytics screens, sourced from Dribbble and Pinterest

> 💡 **The takeaway:** the prompt itself was short — the `context.md` file did all the heavy lifting. A clear, structured spec consistently outperforms a long chain of back-and-forth prompts, and it costs far fewer tokens along the way.
